# 管理员使用指南

系统管理员拥有最高权限，可以管理用户、查看和修改学习数据、调节系统参数。

---

## 1. 角色说明

管理员可以：
- 管理所有用户（查看、角色变更、密码重置、删除）
- 查看和修改**任意用户**的 SM-2 学习状态
- 运行时修改系统配置参数（热加载，无需重启）
- 重新加载教学内容
- 拥有考核者的所有权限（创建/管理考试）
- 查看审计日志（config_changelog 表）

---

## 2. 导航入口

管理员登录后导航栏显示：

| 导航 | 功能 |
|------|------|
| 📊 首页 | 仪表盘 |
| ⚙️ 管理 | SM-2 调试 + 手动覆盖 + 参数调节 |
| 👥 用户管理 | 用户列表、角色变更、密码重置、删除用户 |
| 📝 考试管理 | 创建和管理考试（同考核者） |
| 📋 综合考试 | 查看考试列表（同学习者） |

---

## 3. 管理面板 (`/admin`)

### 3.1 用户选择器

顶部下拉框「👤 查看用户」：
- 默认显示管理员自己的 SM-2 数据
- 切换到任意用户后，下方调试表和覆盖操作**都作用于该用户**
- 鼠标悬停 `?` 图标查看详细说明

### 3.2 SM-2 调试器

展示选中用户的所有章节 SM-2 状态：

| 列 | 含义 |
|----|------|
| 节点 | 章节标题 |
| 学科 | 🧮 数学 / 📖 英语 |
| 掌握度 | 0~1，≥0.85 可解锁后续章节 |
| EF | Easiness Factor，难度系数（默认 2.5，最低 1.3） |
| 间隔 | 当前设定的复习间隔天数 |
| 状态 | locked → unlocked → learning → mastered → degraded |
| 下次复习 | 建议的复习日期 |

**用途：** 排查学生卡在某个章节的原因、确认掌握度是否达标。

### 3.3 手动覆盖

修改指定章节的 SM-2 参数：

| 参数 | 说明 |
|------|------|
| 节点 slug | 章节标识符，从上方调试表复制（如 `factorization`） |
| EF | Easiness Factor，建议范围 1.3~3.0 |
| 掌握度 | 0~1，设为 0.9 可直接标记为已掌握 |
| 状态 | locked / unlocked / learning / mastered / degraded |

**典型场景：**
- 学生卡住 → 将 locked 改为 unlocked
- 误标记 → 将 mastered 降回 learning
- 加快进度 → 提高掌握度至 0.9+

每次修改会写入 `config_changelog` 审计日志。

### 3.4 参数调节

运行时修改 `config.yaml` 参数，**自动热加载无需重启**：

| 参数路径 | 含义 | 默认值 |
|----------|------|--------|
| `sm2.initial_ef` | 新章节初始 EF | 2.5 |
| `sm2.mastery_threshold` | 掌握门槛 | 0.85 |
| `sm2.min_ef` | 最低 EF | 1.3 |
| `app.content_dir` | 内容文件路径 | `./content` |

参数路径用 `.` 分隔。修改写入 `config_changelog` 审计日志。

### 3.5 重新加载内容

修改 `content/nodes/` 或 `content/questions/` 下的 YAML 文件后，点击此按钮立即生效。

## 4. 用户管理 (`/admin/users`)

### 4.1 用户列表

| 列 | 内容 |
|----|------|
| 用户名 | 注册用户名 |
| 角色 | 下拉修改：学习者 / 考核者 / 管理员 |
| 注册时间 | 注册日期 |
| 学习进度 | 已掌握/总章节数及百分比，点击展开节点详情 |
| 考试 | 参加考试次数 |
| 操作 | 密码重置 / 删除用户 |

### 4.2 角色筛选

顶部标签页：全部 / 学习者 / 考核者 / 管理员，快速筛选。

### 4.3 修改角色

直接点击角色下拉框切换，即时生效。

### 4.4 重置密码

点击「🔑 密码」→ 输入新密码（至少 4 位）→ 确认重置。

### 4.5 删除用户

点击「🗑 删除」→ 二次确认弹窗 → 确认删除。系统会自动清理：
- 学习进度 (user_progress)
- 答题记录 (question_attempts)
- 学习记录 (daily_sessions)
- 反思日记 (thought_diaries)
- 考试记录 (exam_attempts, exam_results)
- 聊天记录 (chat_sessions, chat_messages)
- 该用户创建的考试保留（creator_id 置空）

**安全限制：** 不能删除自己的账号。

### 4.6 查看用户进度

点击「学习进度」列的数字，展开该用户每个章节的详细状态（slug、标题、状态标签）。

---

## 5. 命令行工具 (`admin_cli.py`)

紧急情况下（如无法登录 Web 界面）可通过命令行管理：

```bash
# 创建管理员账号
python -m backend.admin_cli create <用户名> <密码>

# 升级已有用户为管理员
python -m backend.admin_cli promote <用户名>
```

---

## 6. 数据库与迁移

- 数据库文件：`data/training.db`（SQLite，文件型持久化）
- 迁移系统：`backend/migrate.py`，版本化管理，不会丢失用户数据
- 当前迁移版本记录在 `_schema_version` 表中

---

## 7. 配置文件

`config.yaml` 控制应用全局参数，支持热加载。管理员通过 Web 界面修改即写入此文件。

`content/` 目录下的 YAML 文件定义所有教学内容，格式说明见项目根目录 `CLAUDE.md`。
